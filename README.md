# WatchZone

## Apresentação da Empresa

Este projeto é dedicado à análise e compreensão da recente onda epidêmica de dengue. A epidemia de dengue é um desafio global de saúde pública, afetando milhões de pessoas em todo o mundo a cada ano. Nosso objetivo é investigar os principais fatores que contribuíram para a propagação dessa doença em nossa região específica, bem como desenvolver estratégias para mitigar seus impactos e prevenir futuras crises.

### Missão

O projeto da nossa empresa foca em desenvolver soluções tecnológicas para prevenir epidemias, com ênfase inicial na dengue. Nosso objetivo é demonstrar como a tecnologia pode ser uma aliada eficaz na área da saúde pública, indo além do ambiente virtual para impactar diretamente na prevenção e controle de doenças.

### Visão

A estratégia de implementação delineada neste documento visa introduzir gradualmente soluções tecnológicas para prevenção de epidemias nos ambientes da saúde pública, começando em nível local e expandindo progressivamente para níveis maiores. Ao colaborar com as autoridades de saúde e outras partes interessadas, esperamos fazer uma contribuição significativa para a melhoria da saúde pública em todo o mundo.

### Valores

1. **Integridade e Ética**: Comprometemo-nos a agir com integridade e ética em todas as nossas atividades, mantendo altos padrões de honestidade, transparência e responsabilidade.
2. **Excelência e Inovação**: Buscamos a excelência em tudo o que fazemos, incentivando a inovação e a busca contínua por soluções criativas e eficazes para os desafios que enfrentamos.
3. **Impacto Positivo**: Priorizamos o impacto positivo em nossos clientes, colaboradores, comunidades e no mundo em geral.

## Escopo do Sistema

O escopo do sistema inclui:

- Análise de dados através de gráficos.
- Login para funcionários públicos.
- Login para usuários.
- Dashboard de controle.

### Objetivos do Projeto

- Controle de dengue de forma visual e interativa.
- Ajuda na área da saúde.
- Conscientização e prevenção.
- Análise de dados.

### Técnica de Levantamento de Requisitos

- **Entrevistas com Stakeholders**: Realizar entrevistas com diferentes partes interessadas para entender suas perspectivas, desafios e necessidades relacionadas ao controle da dengue.
- **Observação Direta**: Observar diretamente as práticas e desafios enfrentados por profissionais de saúde e autoridades locais no combate à dengue.
- **Análise de Documentos**: Analisar documentos existentes, como relatórios de vigilância epidemiológica, registros de casos de dengue e planos de controle de vetores.

### Requisitos Funcionais

1. **Login de Funcionários Públicos**:
   - Funcionalidade de login segura para funcionários públicos.
   - Recuperação de senha para funcionários.
2. **Login de Usuários**:
   - Funcionalidade de login para usuários do sistema.
   - Recuperação de senha para usuários.
3. **Dashboard de Controle**:
   - Visualização de dados epidemiológicos em tempo real.
   - Gráficos interativos mostrando a incidência de casos de dengue.
   - Mapas de calor indicando áreas de alto risco.
4. **Análise de Dados**:
   - Ferramentas para análise de dados históricos de dengue.
   - Geração de relatórios automáticos.
5. **Conscientização e Prevenção**:
   - Seção de dicas de prevenção para a comunidade.
   - Notificações sobre surtos e medidas preventivas.

### Requisitos Não Funcionais

1. **Desempenho**:
   - O sistema deve ser capaz de processar grandes volumes de dados sem perda significativa de desempenho.
2. **Segurança**:
   - Os dados devem ser armazenados e transmitidos de forma segura, com criptografia apropriada.
   - Implementação de autenticação e autorização robustas.
3. **Usabilidade**:
   - A interface do usuário deve ser intuitiva e fácil de navegar.
   - Acessibilidade para usuários com deficiência.
4. **Escalabilidade**:
   - O sistema deve ser escalável para suportar um número crescente de usuários e dados.
5. **Disponibilidade**:
   - O sistema deve ter alta disponibilidade, com um tempo de inatividade mínimo.

## Testes e Qualidade

### Estratégia de Testes

Utilizamos uma abordagem de testes automatizados e manuais para garantir a qualidade do software. Os testes incluem:

- **Testes Unitários**: Garantem que cada componente individual funcione corretamente.
- **Testes de Integração**: Verificam se os componentes interagem corretamente entre si.
- **Testes de Sistema**: Validam o sistema como um todo, incluindo todas as funcionalidades.
- **Testes de Aceitação**: Asseguram que o sistema atende aos requisitos e expectativas dos usuários finais.

### Resultados dos Testes

- **Testes Unitários**: 95% de cobertura, todos os testes passando.
- **Testes de Integração**: 100% de sucesso nas interações críticas.
- **Testes de Sistema**: Validação completa sem falhas.
- **Testes de Aceitação**: Feedback positivo dos stakeholders, com pequenas melhorias sugeridas e implementadas.

### Garantia da Qualidade

- **Revisões de Código**: Todos os commits são revisados por pelo menos um colega de equipe.
- **Integração Contínua**: Implementação de CI/CD com Jenkins para garantir a integração contínua e entrega rápida.
- **Monitoramento e Logging**: Utilização de ferramentas de monitoramento para detectar e corrigir problemas rapidamente.
